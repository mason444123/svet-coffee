# Свет. — сайт, заказы и Telegram-бот

Рабочая версия сайта кофейни с мобильным меню, корзиной, серверной проверкой заказов, SQLite-журналом и закрытым Telegram-ботом для сотрудников.

## Быстрый локальный запуск

Требуется Python 3.11 или новее. Сторонние библиотеки не нужны.

```powershell
Copy-Item .env.example .env
python scripts/generate_bot_pin.py
```

Вставьте полученный `TELEGRAM_ACCESS_PIN_HASH` в `.env`. Затем перевыпустите токен через BotFather и укажите новый токен в `TELEGRAM_BOT_TOKEN`.

```powershell
.\start-server.ps1
```

Сайт откроется по адресу `http://127.0.0.1:8080/`. Проверка сервера: `http://127.0.0.1:8080/api/health`.

## Настройка Telegram

1. Отправьте боту `/start`.
2. Выполните `/login 123456`, подставив сгенерированный PIN.
3. После успешного входа чат сохраняется как авторизованный.

Команды:

- `/orders` — восемь последних заказов;
- `/today` — статистика за сутки;
- `/week` — статистика за семь дней;
- `/month` — статистика за 30 дней;
- `/date 2026-07-21` — статистика за выбранную дату;
- `/profile` — профиль сотрудника и количество обработанных статусов;
- `/order 123` — заказ с контактами и адресом;
- кнопки «Принять», «Готов», «Выдан» и «Отменить» меняют статус заказа.

После входа бот также показывает постоянную клавиатуру: заказы, сегодня, неделя, месяц, профиль и помощь. Новое уведомление содержит выбранный способ получения, способ оплаты, полные имя, телефон, адрес, комментарий и состав заказа.

Можно дополнительно ограничить доступ конкретными Telegram ID через `TELEGRAM_ADMIN_IDS=123,456`. После пяти неверных PIN-попыток вход блокируется на 30 минут.

## Безопасность

- Токен и PIN-хеш находятся только в `.env`; `.env` исключён из Git и релизного архива.
- Сервер не отдаёт `.env`, базу SQLite, `server.py` и каталог `runtime` по HTTP.
- Цены и итог пересчитываются по `data/menu.json`; значения из браузера не считаются доверенными.
- Размер тела запроса, количество позиций и количество заказов с одного IP ограничены.
- Заказы принимает только same-origin запрос с внутренним заголовком клиента.
- В ответах включены CSP, запрет iframe, `nosniff`, ограничение referrer и Permissions Policy.
- Полные контакты показываются только в закрытом чате авторизованного сотрудника.
- Корзина хранится в `sessionStorage`: новая вкладка или новая браузерная сессия начинается с пустой корзины; старые ключи предыдущих версий удаляются автоматически.

Кнопка «Оплатить» создаёт заказ со статусом ожидания онлайн-оплаты. Реальное списание денег требует отдельного договора с банком или платёжным провайдером и его серверных ключей; до подключения эквайринга сотрудник подтверждает наличие и отправляет клиенту платёжную ссылку.

Для публикации нужен HTTPS reverse proxy (например, nginx или Caddy), регулярные резервные копии `runtime/orders.sqlite3` и сервер на территории РФ для первичного хранения данных российских клиентов. Локальный компьютер подходит для проверки, но не для круглосуточного приёма заказов.

## Юридический чек-лист перед публикацией

В `legal.html` уже добавлены рабочие шаблоны:

- политика обработки персональных данных;
- отдельное согласие пользователя;
- сведения о cookie и локальном хранилище;
- условия оформления заказа.

До публикации обязательно замените выделенные поля на реальные реквизиты владельца: полное наименование ИП/организации, ИНН, ОГРНИП/ОГРН, адрес, email и телефон. Также проверьте необходимость уведомления Роскомнадзора об обработке персональных данных и отдельно оцените использование Telegram и возможную трансграничную передачу. Документы следует передать юристу на финальную проверку.

## Настройки сайта

В `assets/config.js`:

```js
window.SVET_CONFIG = {
  orderEndpoint: '/api/orders',
  contactPhoneLabel: '+7 000 000-00-00',
  contactPhoneHref: 'tel:+70000000000',
  openingHours: '08:00–22:00'
};
```

Не помещайте Telegram-токен в `config.js`, HTML или клиентский JavaScript: эти файлы доступны каждому посетителю.

## Проверка

```powershell
node --check assets/app.js
python -m py_compile server.py scripts/generate_bot_pin.py
python scripts/validate_menu_site.py
```

База заказов создаётся автоматически в `runtime/orders.sqlite3` и не входит в архив сайта.
